# 执行步骤

## 一、Agents

### 1.supervisor.py

[1] 作用：编排load_memory-->reasearch-->script-->quality_check-->image-->save-->evaluate保存的全流程，维护graphstate，协调各Agent的输入输出；

[2] 流程：用户传入产品str, 图片路径str, 平台str-->app.invoke(initial_state)-->返回dict(里面有文案list, 图片list)

[3] app构建：get_app-->_app-->build_graph-->初始化graphstate, 创建节点和边load_memory, research, script, image, save, evaluate

[4] load_memory: 接收state, 返回一个dict, 修改其中brand_name的值；

    research: 接收state, 返回一个dict, 修改其中trend_keywords, trend_summary, competitor_styles的值；

    script: 接收state, 返回一个dict, 修改其中scripts的值；

    quality_check: 接收state, 返回一个dict, 修改其中quality_passed, retry_count的值；

    evaluate：接收state, 返回一个dict, 修改其中evaluation_result的值；

    image: 接收state, 返回一个dict, 修改其中image_paths的值；

    save: 接收state, 返回一个dict, 修改其中save_dir的值；

[5] run_pipeline(product_desc: str, platform: str = "小红书", reference_image_path: str | None = None,) -> dict 运行完整生成流程

        Returns:
            {   
                # scripts 是3套文案，天然是列表
                "scripts": list,
                # image_paths 理论上可以生成多张图片，所以也是列表，即使现在只生成1张
                "image_paths": list,
                "save_dir": str,
                "trend_keywords": list,
                "evaluation_result": str,
            }


### 2.research.py

[1] 作用：搜索产品/行业的当前营销趋势、热词、竞品风格；输出：trend_keywords（列表） + trend_summary（摘要）；

[2] 流程：tools-->web_search_tool函数（输入querys str, 调用Tavily搜索）-->返回str，包含摘要和前三条结果；run_research_agent函数-->输入产品名称，平台-->构建llm(deepseek-chat)，
构建agent（传入llm, tools, system_prompt)-->agent.invoke(user_message)-->最终返回dict

[3] 输出格式如下：

    {
        "trend_keywords": ["关键词1", "关键词2", ...],  # 5-8个营销热词
        "trend_summary": "趋势摘要，100字以内",
        "competitor_styles": ["风格描述1", "风格描述2"]
    }

### 3.script.py

[1] 作用：根据产品描述 + 趋势关键词，生成3套不同平台风格的营销文案; 输出：scripts（list of dict，每套含 platform/title/body/hashtags）；

[2] 流程：system_prompt(要求使用COT)-->COT_TEMPLATE-->定义run_script_agent函数

[3] run_script_agent：输入（product_desc: str, platform: str, trend_keywords: list, trend_summary: str, brand_memory: list）-->定义llm, 接收memory, 定义user_prompt, message列表
-->llm.invoke(messages) -->提取json-->返回最终结果dict

[4] 输出格式如下：

    {{
      "scripts": [
        {{
          "platform": "小红书",
          "title": "标题（25字内）",
          "body": "正文内容",
          "hashtags": ["话题1", "话题2", "话题3"]
        }},
        {{
          "platform": "微博",
          "title": "标题",
          "body": "正文内容",
          "hashtags": ["话题1", "话题2"]
        }},
        {{
          "platform": "朋友圈",
          "title": "",
          "body": "朋友圈文案（无标题）",
          "hashtags": []
        }}
      ]
    }}

### 4. quality_agent.py

[1] 作用：规则层文案质量检查，判断Script Agent生成的文案是否达标，控制重试次数；

[2] 流程：run_quality_check函数-->检查文案套数是否为3套、每套body是否超过50字-->返回dict

[3] run_quality_check(scripts: list, retry_count: int) -> dict

检查规则：

必须生成3套文案
每套文案body长度须超过50字
不达标时retry_count自增，supervisor根据retry_count决定是否回退重试
retry_count >= 2时强制放行，避免无限循环
返回格式：


{
    "quality_passed": bool,
    "retry_count": int,
}

### 5.image.py

[1] 作用：1. 若用户上传了参考图，用视觉模型分析风格; 2. 将风格 + 文案摘要拼成 DALL-E 3 prompt; 3. 调用 generate_image 生成图片并下载本地；

[2] 流程：run_image_agent函数-->输入（product_desc: str, scripts: list[dict], reference_image_path: str | None, output_dir: str = "./outputs"）-->分析参考图（如果有）_analyze_reference_image(reference_image_path)
-->构建 prompt（_build_dalle_prompt(scripts, style_desc, product_desc)）-->生成图片generate_image(prompt=dalle_prompt, size="1024x1024", quality="standard")-->返回生成图片的list

[3] _analyze_image_gpt(image_path: str) -> str  -->构建gpt-4o-mini的client(最终传给 gpt-4o-mini 的格式：data:image/jpeg;base64,iVBORw0KGgo...)--> 构建message(使用DALL-E 3进行图片分析)
-->client.invoke([message])-->返回str（"Soft pink tones, centered composition, warm natural lighting, minimalist aesthetic, clean and elegant style"）

说明：这里可以切换qwen，但是由于时间关系，我暂时没有看

[4] analyze_reference_image(image_path: str) -> str -->读取reference_image的路径，根据环境变量的图片分析器的model选择是gpt还是qwen分析图片

[5] _build_dalle_prompt(scripts: list[dict], style_desc: str, product_desc: str) -> str 将文案摘要 + 风格描述合并成 DALL-E 3 prompt-->判断是否是人物/健身/课程类产品，用场景图代替产品图-->判断是否是生活化场景，然后根据不同的prompt生成不同的DALL-E 3prompt（str）

### 6. evaluate_agent.py

[1] 作用：多模态LLM-as-Judge评估，使用通义千问视觉模型（qwen-vl-max）对文案+图片综合评估；与生成模型（DeepSeek）隔离，避免自评偏差；

[2] 流程：run_evaluate函数-->把3套文案拼成文字-->图片转base64-->调用qwen-vl-max多模态评估-->返回dict

[3] run_evaluate(scripts: list, image_paths: list) -> dict

评估维度：文案质量 / 图片合规性 / 图文一致性，每项1-10分

返回格式：


{
    "evaluation_result": str,
}
[4] 设计说明：评估模型与生成模型分离，对应内容审核场景中初审模型和复审模型分离的设计思路。

[6] 启动服务：mcp.run(transport="stdio")

说明：启动 MCP Server，transport="stdio" 表示通过标准输入输出（stdin/stdout）和 Agent 通信。就是 Agent 和 MCP Server 之间通过命令行的输入输出流传递数据，适合本地进程间通信。

对应之前设计时说的两种方式：

stdio：本地进程通信，简单，适合 demo ✅ 用的这个

SSE：HTTP 长连接，适合远程部署

## 四、ui

### app.py

1. 作用：Gradio 前端界面；布局：左侧输入区 | 右侧文案Tab + 图片展示。

2. 代码执行逻辑：

    ui/app.py（用户点击）
        → api/main.py（接收请求）  说明：本地直接调用supervisor，如果启动对外服务，才会用到fastapi的main.py
        → supervisor.py（编排）
        → 各 agent
        → mcp/tools
        → memory

3. 函数1：generate_content(product_desc: str, platform: str, reference_image)  核心回调。Gradio 回调：触发完整生成流程。
   
   返回 return xhs, weibo, pyq, image_paths, status, keywords_str
   
4. UI布局：with gr.Blocks() as demo-->gr.Markdown()-->withgr.Row()-->with gr.Column() (两个)-->generate_btn.click()-->gr.Markdown()-->demo.launch(server_name="0.0.0.0", server_port=7860, share=False)

        启动 Gradio 服务：
        server_name="0.0.0.0" — 监听所有网络接口，局域网内其他设备也能访问
        server_port=7860 — 端口号，浏览器访问 http://localhost:7860
        share=False — 不生成公网链接，只在本地/局域网访问
        如果 share=True 会生成一个临时的公网链接（通过 Gradio 的服务器中转），方便分享给别人演示，但在 Codespace 里不需要，因为 Codespace 自动提供端口转发。
